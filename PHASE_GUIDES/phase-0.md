# Phase 0 — Foundation

**Goal:** Establish the project. Create `PROJECT.md` with project identity, scope, success definition, and team.

**Duration:** One session (or Royce provides direction and I draft in one pass)

## Steps

### Step 0.1 — Project Identity

Write the first draft of `PROJECT.md`:

```markdown
# [Project Name]

**Elevator pitch:** One sentence that explains what this is and why it matters.

**Problem:** What problem does this solve? Who has it?

**Solution:** How does this product solve the problem?

**Target audience:** Primary users. Be specific — not "everyone."

**Unique value:** Why this, why now, why us?
```

### Step 0.2 — Scope Definition

Answer:
- What is explicitly **in scope**?
- What is explicitly **out of scope**?
- What's **MVP scope** vs future phases?

### Step 0.3 — Success Definition

Define what "done" looks like:
- User success metric (e.g., "users can submit benchmarks and see community averages")
- Business metric (e.g., "100 active benchmark submissions in first month")
- Go-no-go criteria (e.g., "verified build page loads in <2s, deployable to Netlify")

### Step 0.4 — Team Roles

- **Product Owner:** Royce — makes decisions, approves phases
- **Developer:** Operant — executes, plans, reports
- **Optional:** Additional specialists (design, dev ops)

## Output

- `PROJECT.md` — complete project foundation
- `PHASE_STATE.md` — Phase 0 complete, Phase 1 queued

## Gate

Royce reviews and approves `PROJECT.md`. Changes allowed. When Royce says "looks good, start Phase 1" — we proceed.
