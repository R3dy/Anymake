# Phase 5 — Launch

**Goal:** Ship the product, validate, iterate.

**Duration:** Ongoing

## Steps

### Step 5.1 — Pre-Launch Checklist

- [ ] All environment variables configured
- [ ] Monitoring/observability in place
- [ ] Error tracking (Sentry or similar)
- [ ] Analytics (page views, signups, key actions)
- [ ] Support channel (email or Discord)
- [ ] Terms of service / privacy policy (if required)

### Step 5.2 — Deployment

- Configure production hosting
- DNS and SSL
- Run smoke test on production URL
- Verify all critical paths work

### Step 5.3 — Beta Release (Soft Launch)

- Limited release (10-50 users, or invite-only)
- Collect feedback actively
- Fix critical bugs
- Measure engagement metrics

### Step 5.4 — Public Launch

- Announce (social, relevant communities)
- Monitor for issues
- Respond to feedback
- Fix non-critical issues in iteration

### Step 5.5 — Iteration

Based on metrics and feedback:
- Prioritize bugs and polish
- Plan next feature set
- Start Phase 6 planning if needed

## Output

- Live product at production URL
- Metrics dashboard (page views, signups, engagement)
- Bug backlog (prioritized)
- Feature backlog (prioritized)

## Gate

Product is live. Phase 5 doesn't "complete" — it's ongoing. Use Phase 5 for maintenance and iteration cycles.

## Post-MVP

After successful launch and initial iteration, start a new cycle:
- Phase 2: Plan new features
- Phase 3: Solution the next epic
- Phase 4: Build it
- Phase 5: Ship it

BMad is a cycle, not a one-time event.
